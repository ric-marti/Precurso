Ahora veremos en detalle como js da dinamismo a una web modificando, incluyendo y eliminando elementos así como ejecutando acciones precedidas de otras acciones.
Antes de nada, necesitamos saber que desde js podremos referenciar cualquier elemento de nuestro HTML, para lo que existen varias formas.

  • document.get.ElementById('idElemento'):

document hace referencia a la pag-web completa, todo el HTML escrito, las etiquetas head, body y el resto se incluyen. La instrucción getElementById('IdElemento') coge un elemento que tenga el id igual que el que le pasamos entre paréntesis. Algo así como "ey, recorre el documento y coge aquel elemento cuyo id coincida con el 'IdElemento'". Esta instrucción genera un objeto a raíz del 'idElemento'.

  • document.getElementByTagName('nombreElemento'):

Coge todos los elementos que compartan el 'nombreElemento' (p, a, h1, div...). A diferencia de la instrucción previa, esta en vez de generar un objeto, genera una lista de objetos recogidos del HTML. La forma de tratar con ella será ligeramente diferente a la anterior.

  • document.getElementsByClassname('classElementos'):

Misma función que el anterior. Recorre el documento HTML y recupera todos los elementos con el mismo classname en su atributo class. También devolverá una lista de elementos HTML.

  • document.querySelector('textoLibre'):

Como en la primera forma, referencia elementos HTML por su id. Al igual que todos los anteriores, nos permite decirle que nos busque por class o por el propio elemento, lo único que con una particularidad que si seleccionamos por id hay que poner '#' delante del nombre, y si es por class '.' (para el elemento con el tipo bastará). Si se usa para coger más de una referencia (>1 elemento o >1 classname) coge la primera coincidencia y no hará una lista como los dos previos.

  • document.querySelectorAll('textoLibre'):

Nos ayuda a recoger todos los elementos de un mismo tipo(class, elemento) , pudiendo usarlo indistintamente por lo que no se necesita tener otros dos métodos para recoger varios elementos presentes.

Revisar el 11_ejemplo_referencia_elemento_html y js para ver y comprobar sus distintas funcionalidades. En la consola y con los comentarios se entenderá bien.
