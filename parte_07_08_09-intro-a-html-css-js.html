Intro a html, css y js

En esta parte se veran las diferencias principales entre html, css y js, nos iniciaremos en la maquetacion web (html + css) y acabaremos dandole funcionalidad con js, manejando lo que se conoce como el DOM (Document Object Model).

HTML

HTML (Hyper Text Markup Languaje o Lenguaje de marcasa de Hiper Texto) es un lenguaje de marcado cuya funcionalidad es indicarle al navegador web el tipo de elemento que está en la estructura de una página web. Es el componente más basico de una web, la estrucura web. Define la forma y lugar de los diferentes componentes de una página web, como el texto, las imágenes, cabezera, links, etc. Algo asi como los cimientos de una casa, una especie de hormigón moldeable. O de otro modo, el esqueleto humano.

Etiquetas: strong/b - em/i

"strong" y "b" tienen la misma utilidad, destacar(strong) el texto poniéndolo en negrita(b). La diferencia entre ambas es simplemente la semántica, ya que b proviene de bold y strong significa fuerte.

"em" e "i", al igual que las dos previas, tienen la misma utilidad pero provienen de diferentes semánticas. em(em) vien de la palabra emphasis(énfasis), mientras que i(i) es de italic. Recomiendan utilizar em antes que i.

Enlaces/Links: "a"

Se escriben bajo la etiquta "a" y llevan atributos. Un atributo es una propiedad que un componente puede manejar y que en ocasiones necesita para poder funcionar bien. Estos siempre se escriben dentro de la etiqueta de apertura, normalmente como clave = "valor". En el caso de los enlaces, basta con poner las etiquetas de apertura y cierre más el texto en el medio. Además, necesitamos indicarle la dirección a la que nos va a redirigir tras clickar en la "a". Para ello la etiqueta "a" posee un abributo llamado href (Hypertext reference) en el que debemos incluir la ruta (direccion URL) que queremos usar. Como por ejemplo este enlace que te lleva al buscador Google. Por cierto, esta es una ruta absoluta, es decir, necesita toda la dirección para que el enlace funcione bien.

Al poner "este enlace" y "Google" dentro de la etiqueta "a" con el atributo "href" han cambiado de color, de estilo. Esto se debe a que hay ciertos elementos, como este, que tienen sus propios estilos al hacer click.

Imágenes: "img"

Las imágenes son un archivo diferente del HTML, por lo que habrá que indicar al navegador que aceda a un archivo, la imagen, y lo carge. A diferencia de los links, que se enlazan con href, las imágenes se enlazan con src (source, origen en inglés). La imagen es parte del contenido (HTML) y no decorativa (CSS).

Las imágenes tienen el atributo alt (texto alterno) que se muestra cuando la imagen no puede cargarse, pero que también sirve para los lectores de pantallas. Se le puede añadir los atributos width (ancho) y height (alto) para determinar su tamaño, pero debe/es mejor hacerse con CSS. Del mismo modo que con los enlaces, se puede marcar su origen de modo absoluto o relativo.

Especificarle un id a un elemento nos sirve para poder acceder a este con CSS para modificar su apariencia o como método de navegación interna, puesto que la a también sirve para movernos a diferentes secciones de la misma página en la que estamos. El DOM (Document Object Model/Objeto Documento Modelo)

El DOM es el plano de nuestra web. Es el mapa que crea el navegador cuando abre nuestra página para saber qué elementos existen en ella, cuál es su organización y qué atributos poseen. Gracias a ello puede mostrar correctamente en el navegador nuestra página con todos los detalles que le habíamos dado. Podéis verlo en el navegador, a través del inspector del propio navegador. CSS

CSS (Cascade StyleSheet u Hoja de Estilos en Cascada) es tambien un lenguaje de marcado y se encarga de dar los atributos visuales a una página web (marcando o no la diferencia) tales como el color, las imágenes, formas, bordes o tipos de letra. Fue creado para controlar el aspecto o presentación de los documentos electrónicos definidos con HTML y XHTML. CSS es la mejor forma de separar los contenidos y su presentación y es imprescindible para crear páginas web complejas.

Separar la definicón de los contenidos y de su aspecto presenta numerosas ventajas, ya que obliga a crear documentos HTML/XHTML bien definidos y con significado completo (también llamados "documentos semánticos"). Además, mejora la accesibilidad del documento, reduce la complejidad de su mantenimiento y permite visualizar el mismo documento en infinidad de dispositivos diferentes.

Al crear una página web pimero se utiliza HTML/XHTML para marcar los contenidos, es decir, para designar la función de cada elemento dentro de la página: párrafo(p), titular(h), texto destacado(b), tabla(t), lista de elementos(ol/ul + li), etc. Después se utiliza CSS para definir el aspecto de cada elemento: color, tamaño, tipografía, separaciones (horizontal y vertical) entre elementos, posición de cada elemento dentro de la página, etc.

CSS se puede usar de varias formas, ya sea mediante estilos en línea (no recomendado por la limpieza del código) o mediante archivo CSS.

Estilos en línea

Este tipo de elemenos son fáciles de identificar ya que la propiedad style destaca dentro de la etiqueta de apertura (en este caso "p"), pero si uno quiere que todos los párrafos tengan un color y tamaño determinado y el día de mañana hay que cambiarlo... Sintaxis

Los archivos CSS tienen la terminación archivo.css y para usarlos se necesita poner dentro de la etiqueta head del HTML un elemento link para asociar o linkear ambos archivos, el .css con el .html: link rel="stylesheet" href="archivo.css"

Hay tres formas principales de establecer los estilos con los que definir los elementos HTML:

  • Referenciar las etiquetas (que sean todas iguales): Sintaxis: nombre de la etiqueta(p,h3,li...) {propiedades a modificar(color, font-size...): valores(red, 20px...)}
  • Utilizando la propiedad id: Sintaxis: nombre del id{propiedades a modificar(color, font-size...): valores(red, 20px...)}
  • Utilizando el atributo class, agrupando varios y diversos elementos para darles un estilo en particular. Sintaxis: nombre del atributo class{propiedades a modificar(color, font-size...): valores(red, 20px...)}. class se utiliza cuando queremos agrupar varios.

Especificidad y su Jerarquía

Qué estilos se aplicarían si se usasen los tres (o cuatro) tipos de sintaxis de forma simultánea? La especifidad es la manera mediante la que los navegadores deciden qué valores de una propiedad CSS son más relevantes para un elemento, y por lo tanto, la que en base a las regras de coincidencia compuestas por diferentes tipos de selectores CSS se aplicará. Si varias tienen igualdad de especificidad se aplicará la última declaración encontrada en el CSS. El orden es el siguiente:

  1. Selector de tipo (p, div, h1...).
  2. Selector de classe (.orange, .green...).
  3. Selector de id (#title...).
  4. Estilos en línea

!important (exclamación(!) + important) sobreescribe cualquer declaración de estilos. No tiene relación con la especificidad pero influirá/interactuará directamente con esta. Su uso debe ser limitado a sobreescribir estilos de librerías externas. NO USAR POR EL MOMENTO

Ejemplos de Especificidad

Algunos ejemplos de Especifidad utilizando los mismos elementos (h1 y p) varias veces con la especificidad de los selectores modificada para que se vea la diferencia. Esta capacidad de decisión sobre que estilos utilizar en qué elementos le da el nombre de "Hoja de Estilos en Cascada", pues va aplicado estos desde lo más genério a lo más específico.

!important rompe esa cascada. Su uso debe ser muy limitado puesto que hay que aplicarlo en varios elementos para modificar uno solo.

  1. Sin selectores (id, class) ni estilos en línea. Aquí se aplicarán los estilos que traen por defecto los elementos html.
  2. Con selectores class sin estilos en línea. Aquí se aplicarán los estilos que trae la class porque tiene mayor especifidad.
  3. Con selecores(id, class) sin estilos en línea. Aquí se aplicarán los estilos que trae el id porque tiene mayor especificidad.
  4. Con selectores (id, class) con estilos en línea. Aquí se aplicarán los estilos en línea poruqe tiene mayor especificidad.

Permite la ubicación de elementos internos de un contenedor (Ej.: div) de forma dinámica, alterando su posición con respecto a su comportamiento (en bloque vs en línea) y su propio lugar en el contenedor.

Java Script (JS)

JavaScript (que no Java) es el lenguaje de programación que (mayoritariamente) da funcionalidad a una página web. Es el músculo detrás de las webs. Mientras un boton se hace con HTML y CSS da la apariencia (color, tipo de letra, bordes...), es JS el que da dinamismo a la página web, modifi cando, incluyendo y eliminando elementos; ejecutando acciones cuando suceden cierta acciones (valga la redundancia) hace que se ejecuten las acciones, cambios y modificaciones dinámicas, que sucedan cosas en los elementos HTML y/o estilos CSS al ejercutar alguna acción como que cuando el usuario haga click en un botón dé una orden para que, por ejemplo, pueda logearse en una página web o cualquier otra acción.

Hasta ahora hemos visto cómo se declaran variables, bucles y condicionales, operadores lógicos y ternarios, arrays y sus métodos y strings y sus métodos. Hemos recogido valores por prompt y los hemos transformado (o no) a números. Hemos usado console.log para mostrar los resultados de nuestro código en la consola del navegador, y hemos puesto todo nuestro código Javascript entre dos etiquetas "script" dentro de archivo HTML.

¿Qué veremos ahora?

  • Los objetos, que en js son "variables" algo más complejas con determinada organización interna. Navegaremos por ellas para llegar al valor que queremos conseguir.
  • Manipular el DOM con js, cambiando elementos de la pagina web de manera dinámica conforme van cierta acciones.
  • Pondremos el códgio de JS en su porpio archivo, uniéndolo al HTML de forma similar como se hace con un archivo de CSS.

Objetos en JavaScript

La Programación Orientada a Objetos es un tipo de programación en el que se toman elementos del mundo real (objetos) y se utilizan de ellos aquellas partes o atributos que necesitamos para que nuestro programa funcione. Ejemplo.: si tengo un programa para pintar fuselajes de avión, no me es relevante el tipo de motor, material de las ventanillas ni la compañia. Lo que me interesa, lo que me influye, es la marca y el modelo para tener en cuenta su medidas y capacidades de carga. Todo eso para poder calcular cuánta pintura necesitaré para ese avión.
